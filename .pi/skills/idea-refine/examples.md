# Ideation Session Examples

## Example 1: Vague Early-Stage Concept

**User:** "I want to help small local restaurants compete with big delivery platforms"

**Phase 1 — Restate:** "How might we give independent restaurants the reach and convenience customers expect from delivery platforms, without forcing them into a model that erodes their margins and brand?"

**Questions reveal:** Restaurant owners. Core pain is 25-30% commission. The problem is customer dependency, not delivery logistics.

**6 Variations generated:**
1. **Direct Channel Toolkit:** Branded ordering (web, SMS, app) for existing customers
2. **Inversion — Charge Customer:** Customer pays small premium, restaurant pays zero
3. **Constraint Removal — Pickup-First:** Ordering + pickup only, no delivery
4. **Combination — Restaurant Collective:** 20-50 restaurants share platform, split costs
5. **Audience Shift — For Regulars:** Loyalty/reorder tool for existing regulars only
6. **Expert Lens — Zero Restaurant Effort:** Auto-texts regulars, auto-posts specials, runs itself

**Phase 2 — Converge:** Variations 5+6 combined = "Regulars Engine". Core bet: top 20% customers drive 80% revenue. Every direct reorder saves 25-30% commission.

**Phase 3 — One-Pager:** "ReOrder: Keep Your Regulars Ordering Direct" — SMS-first, pickup-only MVP, zero restaurant management.

## Example 2: Feature Within Existing Product (Codebase-Aware)

**User:** "Adding real-time collaboration to our document editor"

**Context from codebase:** Block-based editor architecture with REST API. No WebSocket infrastructure.

**5 Variations:**
1. Block-Level Locking (simplest)
2. Async-First Collaboration (inversion)
3. Presence Without Co-Editing (simplification)
4. Collaboration + Version Branches (combination)
5. AI-Mediated Collaboration (10x)

**Recommendation:** #1 or #3 — need to ship to close competitive deals, simplest version that checks the box.

## Example 3: Process/Workflow Idea

**User:** "Better way to run weekly team retrospectives"

**Diagnosis:** Action item problem — nothing changes after retros, so people disengage.

**6 Variations:**
1. Fix the Output (one action item max, demo next week)
2. Kill the Meeting (async written retro)
3. Retro for the Quiet Ones (anonymous, dot-voting)
4. One Question Retro (different question each week)
5. Retro + Experimentation (treat output as hypothesis)
6. Team Health Check (anonymous safety score)

**Recommendation:** Start with #1 (zero-cost experiment) combined with #3 (anonymous submission).

## What to Notice

1. The restatement changes the frame
2. Questions diagnose before prescribing
3. Variations have reasons, not just labels
4. Phase 2 is honest — weak ideas get called out
5. The output is actionable, not theoretical
6. The "Not Doing" list does real work
7. The skill adapts to context (product, feature, process)
