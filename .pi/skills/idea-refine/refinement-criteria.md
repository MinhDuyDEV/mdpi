# Refinement & Evaluation Criteria

Use this rubric during Phase 2 (Evaluate & Converge) to stress-test idea directions.

## Core Evaluation Dimensions

### 1. User Value

**Painkiller vs. Vitamin:**
- **Painkiller:** Solves an acute, frequent problem. Users actively seek it out and will switch.
- **Vitamin:** Nice to have. Makes something marginally better. Users won't change behavior.

**Questions:** Can you name 3 specific people with this problem right now? What are they doing today instead? Would they switch? Is this a "pull" or "push" problem?

### 2. Feasibility

**Technical:** Does the core technology exist? What's the hardest problem? Dependencies on third parties?

**Resource:** Minimum team for MVP? Specialized expertise needed? Regulatory requirements?

**Time-to-value:** How quickly can something reach users? Is there a version deliverable in days/weeks?

### 3. Differentiation

Types (strongest to weakest):
1. New capability (previously impossible)
2. 10x improvement
3. New audience
4. New context
5. Better UX
6. Cheaper (weakest)

**Question:** If a user described this to a friend, what would they say? Is that compelling?

## Assumption Audit

For every direction, list:

- **Must Be True (Dealbreakers):** If wrong, kills the idea. Validate first.
- **Should Be True (Important):** Significantly impacts success but doesn't kill the idea.
- **Might Be True (Nice to Have):** Secondary. Don't validate until core is proven.

## Decision Framework

|                    | High Feasibility | Low Feasibility |
|--------------------|-------------------|-----------------|
| **High Value**     | Do this first     | Worth the risk   |
| **Low Value**      | Only if trivial   | Don't do this    |

## MVP Scoping Principles

1. **One job, done well.** Not three jobs partially.
2. **The riskiest assumption first.**
3. **Time-box, not feature-list.**
4. **The 'Not Doing' list is mandatory.**
5. **If it's not embarrassing, you waited too long.**
